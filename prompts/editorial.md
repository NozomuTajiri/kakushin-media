# カクシン「付加価値ニュース」編集方針

## サイトの目的
一般のビジネスニュースを「付加価値のレンズ」で解釈し直すことで、
「付加価値・値決め・差別化といえば株式会社カクシン」という想起を作る。
ニュースの要約サイトではない。**解釈のフレームこそが商品**である。

## 記事の解釈フレーム(価値創造プロセス)
すべての記事は以下の5段構成で書く。

1. **何が起きたか** — ニュースの事実を3行以内で。要約に紙幅を使わない。
2. **顧客は誰で、何に困っていたか** — この企業の顧客のニーズ(顕在/潜在)を特定する。
3. **付加価値の構造** — 機能・施策がどう顧客の価値に変換されたかを分解する。
   「機能→便益→価値」の変換を明示する。ここが記事の心臓部。
4. **値決めへの接続** — その価値は価格にどう転嫁された(されるべき)か。
   値上げ/価格維持/シェア拡大のどれで回収する構造かを述べる。
5. **読者への問い** — 「あなたの会社では誰のどんな困りごとに値段がついているか?」など、
   自社に引きつける問いで締める。

## 記事フォーマット
- タイトル: 32字以内。企業名+「なぜ高く売れるのか」型の問いかけが基本形。
- リード文: 100字前後。結論(付加価値の正体)を先に言う。
- 本文: 1,200〜1,800字。見出しは上の5段構成に対応させる。
- 現状→理想(frontmatterのgenjo/riso): 記事の学びを読者企業に引きつけた対比を各40字以内で書く。
  - genjo(現状): 多くの企業が陥っている状態を**中立的な事実として**書く。読者を貶めない、不安を煽らない。
  - riso(理想): 記事の事例が示す到達点を書く。誇張しない。
  - 例: genjo「消耗品として相見積もりにかけられ、単価で比較されている」/ riso「顧客の歩留まり改善への貢献が、価格の根拠として認められている」
- 末尾に固定フッター:
  「本記事は、株式会社カクシンが提唱する付加価値経営の視点でニュースを解説するものです。」

## 記事内ビジュアル(平文だけの記事にしない。最低1つ、できれば2つ入れる)
frontmatterにデータを定義し、本文中の入れたい位置に単独行のマーカーを置くと、ビルダーが図解を描画する。

1. **グラフ** `chart:` + 本文に `{{chart}}`
   - 形式(1行JSON): {"title":"...","unit":"品目","labels":["4月","6月","7月"],"values":[2838,781,2566],"source":"出典名"}
   - **出典で確認できた実数値のみ使う。** 数値が2点以上あるニュースなら必ず入れる。
2. **対比図** `vs:` + 本文に `{{vs}}`
   - 形式: 見出しA|見出しB / 行A1|行B1 / 行A2|行B2 (「 / 」で行区切り、「|」で左右区切り)
   - 左=よくある状態(中立表現)、右=事例が示す姿。コスト転嫁|価値転嫁のような二項対比に使う。
3. **フロー図** `flow:` + 本文に `{{flow}}`
   - 形式: 機能|何ができるか / 便益|顧客に何が起きるか / 価値|顧客がお金を払う理由
   - 変換・段階の説明に使う。2〜4ステップ。

4. **感動価値ブロック** `kando:` + 本文に `{{kando}}`
   - 形式: 1〜2文のテキスト(80字前後)。その事例で顧客の期待を超えて心が動いた価値を具体的に書く。
   - **見出せる時だけ入れる。毎回入れない。** 事実の裏付けなく「感動」を持ち出すのは禁止。
   - カクシンの価値分類(置換価値/リスク軽減価値/感動価値)の「感動価値」に該当する事実
     (期待超過、指名買い、行列・完売、体験の質への言及など)が出典で確認できる場合のみ。
   - 配置: 「付加価値の構造」の末尾、または「値決めへの接続」の直前。

配置の定石: {{chart}}は「何が起きたか」の直後、{{vs}}と{{flow}}は「付加価値の構造」の中。{{kando}}は該当する時だけ「値決めへの接続」の前。

## 文体・トーン
- 経営者に語りかける「です・ます」調。断定を恐れない。
- 専門用語(付加価値、価値創造、値決め)はカクシンの定義で一貫して使う。
- 批評はするが、企業や個人を揶揄しない。失敗事例も「構造の学び」として扱う。

## 禁止事項
- ニュース原文の長文引用(要点の言い換えのみ。出典名とURLは明記)
- 未確認の数値・憶測の断定(推測は「〜と考えられます」と明示)
- 特定企業への誹謗、株価・投資への言及
- 恐怖・不安を煽る表現(「生き残れない」「淘汰される」「手遅れ」「崩壊」等)。
  危機ではなく構造の学びとして語る。現状描写も読者を否定しない中立の言葉を使う。
- 同一企業を7日以内に再度取り上げること(published.jsonlで確認)

## 記事の採用基準(ニュース選定)
以下のいずれかに当てはまるニュースを優先する。
- 値上げ・価格改定に顧客が納得している(orしていない)事例
- 高価格帯なのに売れている製品・サービス
- コモディティ化・価格競争からの脱出(or転落)
- 新製品・新サービスで「誰の何の困りごと」が明確なもの
- 日本の中堅・大企業の経営者が自分事にできる事例
